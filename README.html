pptx2js

将 PowerPoint .pptx 文件转换为可直接运行的 PptxGenJS 生成脚本，并附带媒体资源与转换报告。

设计文档 · Markdown 版

运行环境Node.js ≥ 18 版本v0.4.3 许可MIT 仓库GitHub 开源

核心哲学

「PptxGenJS 能力边界即转换边界」

层级策略示例
精确转换 PptxGenJS 原生支持的元素一比一映射 文本框、图片、基本形状、线条、纯色背景
退化转换 尽力保留内容，降级为近似表示 渐变→纯色、未知形状→矩形
静默跳过 记录日志，不中断流程 图表/表格框架、ActiveX、OLE、VBA

功能特性

  • 输入 .pptx，输出 output.js + media/ + conversion.log + 自动生成说明
  • 同时提供 CLI 与 编程 API，共用同一套核心逻辑
  • 转换报告为 JSON，含 slideIndex、elementBounds、severity，便于 CI 集成
  • 仅支持 OOXML（.pptx），不做双向转换、不做 GUI

当前转换能力（v0.4.3）

元素状态说明
文本框（段落对齐、lvl、段距/行距 spcPct、列表、超链接）精确addText()；lstStyle/defRPr 继承
图片精确addImage()；media/ 重名自动后缀
表格（含单元格边框）精确addTable()；text-utils.js
图表 BAR/LINE/PIE/AREA/DOUGHNUT/SCATTER/RADAR/BUBBLE精确addChart()；散点图 c:xVal
预设形状 / 线条（虚线、flipH/flipV）精确addShape()；spTree 文档顺序
母版/版式装饰形状精确master → layout → slide 层叠
纯色 / 渐变 / prstClr / sysClr 背景精确p:bgPr、p:bgRef；系统色修饰符
母版/版式占位符继承精确xfrm、txBody/lstStyle 合并
组合形状（p:grpSp）精确无 p:spTree 时直接展平
SmartArt退化文本列表或占位
渐变填充 / 弯曲连接线退化首色标；bentConnector3→直线
不支持图表类型退化缓存图或占位文本
复杂动画计划design.html

安装

$npm install pptx2js

本地开发：

$git clone https://github.com/yuese12333/pptx2js.git
$cd pptx2js
$npm install
$npm test

使用方式

CLI

$npx pptx2js input.pptx -o ./pptx2js-output

常用参数：

参数默认值说明
input.pptx必填源文件路径
-o, --output./pptx2js-output输出目录
--no-media—不提取媒体，图片引用变为占位注释
--strict-degradefalse任意退化项触发非零退出码
--strict-skipfalseseverity:error 跳过项触发非零退出码
--log-levelinfominimal / info / verbose
--max-file-size50MB超过阈值抛出错误（避免静默 OOM）
Commander 的 --no-media 对应内部选项 media（默认 true）。传入 --no-media 后才会跳过媒体提取。

编程 API

const { convert } = require('pptx2js');

const result = await convert('./input.pptx', {
  outputDir: './pptx2js-output',
  logLevel: 'info',
});

console.log(result.scriptPath);  // .../output.js
console.log(result.logPath);     // .../conversion.log
console.log(result.log.statistics);

运行生成脚本

output.js 依赖 PptxGenJS，需在输出目录单独安装：

$cd pptx2js-output
$npm init -y
$npm install pptxgenjs
$node output.js   # 生成 output.pptx

输出目录结构

pptx2js-output/
├── output.js          # 主生成脚本，可直接 node 运行
├── media/             # 提取的图片等媒体
├── conversion.log     # JSON 格式转换报告
└── README.md          # 自动生成的说明

conversion.log 主要字段：source、statistics、slides[]、degraded[]、omitted[]、warnings[]。

系统架构

六模块流水线（顺序执行）：

input.pptx
① 解压与索引 — lib/unpacker.js
② XML 预解析 — lib/xml-parser.js + lib/rels.js
③ 实体提取器 — extractor, placeholder, table, chart, smartart
④ 映射引擎 — lib/mapper.js
⑥ 资源打包器 — lib/packager.js（先于代码生成，更新 IR 中 mediaPath）
⑤ 代码生成器 — lib/codegen.js
output.js + media/ + conversion.log

辅助模块：lib/presentation.js、lib/graphic.js、lib/xml-utils.js、lib/color.js、lib/bounds.js、lib/text-utils.js、lib/run-utils.js、lib/table.js、lib/chart.js、lib/smartart.js

XML 节点模型（OONode）

自研 lib/xml-parser.js，节点结构 { tag, attrs, children, text }。childNodes 保留文档顺序（Z 轴与段落内 pPr/r 顺序）。

技术栈

用途选型
ZIP 处理JSZip
XML 解析自研 lib/xml-parser.js（无 xml2js 运行时依赖）
CLICommander.js
终端输出chalk
测试Jest（单元 + 集成）

代码格式化在生成阶段自实现缩进拼接，不引入 Prettier。

已知局限

  • L1字体依赖运行环境，不负责检测或打包嵌入字体
  • L2母版继承已实现装饰层与 lstStyle/defRPr；复杂列表/多主题样式仍可能不完整
  • L3SmartArt 仅文本列表退化，缓存图片因 PPT 版本差异未实现
  • L4动画尚未转换（设计为退化淡入，待实现）
  • L5不保证往返 PPTX 二进制一致，追求视觉可接受
  • L6不支持密码保护或 .ppt 旧格式
  • L7不支持增量转换，每次全量重写
  • L8大文件超过 --max-file-size 会直接报错，需调高阈值或拆分
  • L9非标 OOXML无 XML 前缀的文档可能解析失败

项目状态

当前为 v0.4.3：母版/版式装饰层按 Z 轴叠加；lstStyle/defRPr 与 prstClr/sysClr 颜色；spcPct 段距；形状 flipH/flipV；lib/utils 合并为 lib/color.js、lib/bounds.js。复杂动画、部分连接器与 SmartArt 图形化按 design.html 继续推进，欢迎贡献。
